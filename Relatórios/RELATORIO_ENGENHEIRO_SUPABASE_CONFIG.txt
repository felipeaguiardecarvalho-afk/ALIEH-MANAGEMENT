2. APP STREAMLIT: SÓ POSTGRES
--------------------------------------------------------------------------------

  • ``database.connection.get_db_conn()`` devolve **sempre** uma ligação ``psycopg``.

  • Se ``get_postgres_conn()`` falhar (rede, credenciais, pooler, etc.), a app
    **levanta** ``ConnectionError("PostgreSQL connection required but failed")`` após log
    ``FATAL: PostgreSQL connection failed — no fallback allowed``. **Não** há mudança automática
    para SQLite.

  • ``database.config.get_database_provider()`` continua a devolver ``sqlite`` ou ``postgres``
    consoante env (útil para ``init_db``, export, testes). Isso **não** faz a app abrir SQLite
    através de ``get_db_conn``. Sidebar **Backup do banco (admin)** / download de ``.db`` só
    aparece quando ``get_db_provider() == "sqlite"``.

4. POSTGRES — LIGAÇÃO E POOLER (SUPABASE / PGBOUNCER)
--------------------------------------------------------------------------------

  • ``get_postgres_conn``:
      — DSN: ``get_database_url()`` (env DATABASE_URL + segredos) depois ``SUPABASE_DB_URL`` / cadeia;
      — ``sslmode=require`` no URL quando omitido (mantém ``verify-full`` se já definido);
      — ``prepare_threshold=0`` no connect e reforço ``conn.prepare_threshold = 0``;
      — ``autocommit=True``, ``row_factory=dict_row``, ``connect_timeout`` defeito **10** s
        (override ``DATABASE_CONNECT_TIMEOUT``);
      — resolução IPv4 ``hostaddr`` quando aplicável;
      — **Nenhum** ``DISCARD ALL`` nem outros comandos SQL de sessão **após** ``connect``;
      — ``_wrap_postgres_cursor_binary_false``: apenas ``cursor(..., binary=False)`` no cliente;
      — primeira ligação OK (não silent): INFO ``PostgreSQL connection established (PgBouncer safe mode)``.

  • **Consultas:** preferir ``db_execute`` / ``run_insert_returning_id`` (cursor + ``prepare=False``).
    Evitar ``Connection.execute`` em código novo.

  • ``health_check`` probe: ``get_postgres_conn(silent_probe=True)`` + ``SELECT 1`` com ``prepare=False``.

  • **SQLite** (ficheiro): só para ``init_db`` (quando DB_PROVIDER=sqlite), export ``export_all_data``,
    migrações e testes — ``sqlite_connect_for_local_schema(DB_PATH)``. Pastas: ``/data``,
    ``.alieh_data`` ou tempfile; backups podem usar destino alternativo se ``/backups`` falhar.

  • schema_apply: cursor + ``prepare=False`` por instrução; CLI ``--full-reset`` recria ``public`` a partir
    de ``schema.sql`` + grants Supabase (API roles); ver ``python -m database.schema_apply --help``.
    DDL pesado: DSN directo :5432.

  • migrate_sqlite_to_postgres: caminhos absolutos; --full-reset opcional (cópia de dados).
